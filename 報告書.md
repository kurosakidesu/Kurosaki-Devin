本記録は、AI開発エージェント **Devin** を利用して、Excelで作成された画面設計書から業務画面を実装し、その動作確認およびテスト報告書作成までの一連の作業を検証した際の記録である。  

対象となる画面は **移送オーダ一覧画面（OFFSITE-V-0401）** であり、Excel画面設計書を入力として、Devinに対して段階的にプロンプトを与えながら実装・検証を進めた。

# 1. 初回プロンプト（画面実装）

最初にDevinへ与えたプロンプトは、Excelの画面設計書を基に **移送オーダ一覧画面を実装すること** を目的としたものである。

このプロンプトは比較的詳細に制約条件を定義した構造になっており、AIが不要な機能を追加することを防ぐ設計となっていた。

## 主な制約

- 実装対象を **単体画面に限定**
- **他画面の実装は禁止**
- 画面遷移先は **ダミー処理またはTODO**
- **システム全体構築は禁止**

さらに補完ルールも厳密に指定した。

## 補完可能範囲

- 画面項目
- イベント一覧
- データIO
- 画面イメージ

## 追加禁止機能

以下の業務機能は明示的に追加禁止とした。

- 新しい業務ステータス
- 新しい検索条件
- 新しい画面ボタン
- 承認機能
- 通知機能
- バッチ処理
- 帳票出力
- 外部システム連携

# 2. データ設計の指定

データ周りについても以下を明確に指定した。

- 本番DB接続は不要
- SQLiteによる **ローカル仮DBを使用**
- 起動時に **初期データを自動投入**
- テーブル・マスタ・サンプルデータを作成

## サンプルデータ条件

画面確認をしやすくするため以下のデータを含めるよう指定した。

- ステータス違い
- 日付違い
- マスタ未設定
- 空値
- 長い文字列
- 0件検索条件

また、一覧確認のため **約20件のデータを投入**するよう指示した。

# 10. 発生したトラブル

Word報告書が生成されたが、クリックすると以下のエラーが発生した。
NoSuchKey
The specified key does not exist.


原因として考えられるのは

- docx本体が存在しない
- 添付URLが失効
- 添付保存失敗

つまり

**docx生成が成功したように見えて実体が存在しない状態**

---

# 11. 改善対応

添付方式をやめて **repo保存方式** に変更した。
docs/report.docx


指示内容
報告書をリポジトリの docs/report.docx に保存してください。
docs ディレクトリが存在しない場合は作成してください。
