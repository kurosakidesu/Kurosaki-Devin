# 移送オーダ一覧画面 (OFFSITE-V-0401)

Excel画面設計書に基づいた「移送オーダ一覧画面」の実装です。

## 技術スタック

- **フロントエンド**: React + TypeScript + Vite
- **バックエンド**: FastAPI (Python)
- **データベース**: SQLite (ローカル仮DB、起動時に自動作成・初期データ投入)

## 起動手順

### 前提条件

- Python 3.11+
- Node.js 18+
- Poetry

### 1. バックエンド起動

```bash
cd backend
poetry install
poetry run uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

起動時にSQLiteデータベースが自動作成され、20件程度のサンプルデータが投入されます。

### 2. フロントエンド起動

```bash
cd frontend
npm install
npm run dev
```

ブラウザで http://localhost:5173 にアクセスしてください。

## APIエンドポイント

| メソッド | パス | 説明 |
|---------|------|------|
| GET | `/api/orders` | 検索条件付きオーダ一覧取得 |
| GET | `/api/orders/auto-refresh` | 自動更新用オーダ一覧取得 |
| GET | `/api/orders/{internal_no}/{slip_branch_no}/batches` | 割当可能バッチ一覧取得 |
| POST | `/api/orders/{internal_no}/{slip_branch_no}/register-batch` | バッチ登録 |
| GET | `/api/orders/export-csv` | CSV出力 |

## テーブル定義

| テーブル名(論理) | 物理名 | CRUD |
|---|---|---|
| 移送オーダ | transfer_order | RU |
| 移送実績 | transfer_record | RU |
| 出荷ステータスマスタ | m_shipping_sts | R |
| 名称・区分マスタ | m_code_name_pair | R |
| バッチマスタ | m_bat | R |
| バッチ管理品名マスタ | m_bat_item | R |
| ロック中バッチ | locking_bat | CR |

## サンプルデータ

以下のパターンを含む20件のデータを用意:

- ステータス違い (予約/バッチ割当/DCS送信/DCS再送/移送中/一時停止/実績保留/実績送信完了)
- 日付違い (当日/前日/翌日)
- マスタ未設定相当 (管理品名コード・管理品名が空)
- 長い文字列 (レギュラーガソリンプレミアムスーパーロング品名テスト)
- 0件検索になる条件 (翌日以降の日付指定)
